## 模块化输出

当用户只问某一个模块时，可单独输出对应部分：

### 仅问方位吉凶

```
## [城市]方位吉凶表

| 方位 | 板块范围 | 五行属性 | 吉凶等级 | 性价比评级 |
|------|----------|----------|----------|------------|

[各方位简要分析]
```

### 仅问小区推荐

```
## 高性价比小区推荐

### 第一梯队：[方位]（[五行优势]）
| 板块 | 小区举例 | 五行优势 | 价格区间 | 适配理由 |
|------|----------|----------|----------|----------|

### 第二梯队：...
### 第三梯队：...
```

### 仅问住址优化

```
## 住址优化方案：[住址名称]

### 问题分析
[楼层五行问题 + 周边环境五行问题]

### 化解方案
| 位置 | 做法 | 目的 |
|------|------|------|

### 行动清单
1. [立即可做]
2. [本周内做]
3. [长期优化]
```

### 仅问公司优化

```
## 公司优化方案：[公司地址]

### 问题分析
[区位五行 + 楼层五行问题]

### 改造方案
| 位置 | 做法 | 目的 |
|------|------|------|
```

### 仅问搬迁规划

```
## 搬迁优先级规划

### 流年注意事项
| 年份 | 干支 | 五行 | 宜忌 |
|------|------|------|------|

### 优先级行动表
| 优先级 | 行动 | 时间建议 | 理由 |
|--------|------|----------|------|
```

## 注意事项

1. 表格必须用 Markdown 格式，确保对齐
2. 每个表格后附简要分析说明
3. 最终必须有「一句话总结」收尾
4. 所有建议需标注成本级别（零成本/低成本/大额支出）
5. 时间建议需结合流年分析给出具体年份
